# API Reference

Issue Hunter exposes a standard REST API and a WebSocket endpoint using FastAPI.

## REST Endpoints

### `POST /api/hunt`
Starts a new Hunt workflow in a background thread.
- **Body:**
  ```json
  {
    "repo_url": "https://github.com/owner/repo",
    "issues": "1",
    "github_token": "ghp_...",
    "llm_provider": "Google Gemini",
    "llm_api_key": "sk-...",
    "model": "gemini-1.5-pro"
  }
  ```
- **Returns:** `{ "status": "started", "hunt_id": "<UUID>" }`

### `POST /api/benchmark`
Enqueue a batch of repositories for benchmarking.
- **Body:** Array of JSON objects specifying repo URLs and issue arrays.
- **Returns:** `{ "status": "enqueued", "batch_id": "<UUID>" }`

### `GET /api/hunts`
Retrieves a history of all executed hunts.
- **Returns:** List of hunt objects from SQLite, including statuses (`success`, `failed`, `running`).

### `GET /api/hunts/{hunt_id}/logs`
Retrieves the raw terminal logs for a specific hunt.
- **Returns:** `{ "logs": "..." }`

### `GET /api/approvals`
Used by the frontend Diff Viewer to fetch the `git diff` generated by the agent.
- **Query Param:** `hunt_id`
- **Returns:** `{ "diff": "--- a/file.py\n+++ b/file.py\n..." }`

### `POST /api/approvals`
Used by the frontend to submit a human decision.
- **Body:** `{ "hunt_id": "<UUID>", "approved": true }`
- **Returns:** `{ "status": "recorded" }`

### `POST /api/webhook/github`
Listens for GitHub issue comments (e.g., `@issue-hunter fix this`) to automatically trigger a workflow.
- **Headers:** Expects `X-GitHub-Event` and signature validation.

## WebSocket

### `WS /ws/stream`
Streams real-time updates to the UI terminal.
- Emits standard log strings.
- Emits special signal strings like `[APPROVAL_REQUIRED]` to trigger UI state changes.
